===================================
BACKEND TERMS - GET, POST, RENDER, SEND ETC
===================================

Ye do alag categories hain: HTTP Methods aur Response Methods.

-----------------------------------
1. HTTP METHODS (GET, POST, PUT, DELETE)
-----------------------------------
Ye batate hain client kya karna chahta hai server se - request ka type.

GET     - Data fetch/read karna (e.g. homepage load, user list dekhna)
POST    - Naya data create karna (e.g. signup form submit, new post banana)
PUT/PATCH - Existing data update karna (e.g. profile edit karna)
DELETE  - Data remove karna (e.g. post delete karna)

Example code:
app.get('/users', (req, res) => { ... })        // read
app.post('/users', (req, res) => { ... })       // create
app.put('/users/:id', (req, res) => { ... })    // update
app.delete('/users/:id', (req, res) => { ... }) // delete

Ye route define karte time use hote hain - "is URL pe is method ki request aaye to ye function chalao."

-----------------------------------
2. RESPONSE METHODS (render, send, json, redirect)
-----------------------------------
Ye batate hain server client ko kya wapas bhejega - response ka type. 
Ye res object ke methods hain.

res.send()     - Plain text ya simple response bhejta hai. Quick testing, simple string/HTML ke liye.

res.render()   - EJS template ko HTML mein convert karke bhejta hai. Jab dynamic HTML page dikhana ho.

res.json()     - JSON data bhejta hai. APIs ke liye - jab frontend ko sirf data chahiye, HTML nahi.

res.redirect() - Client ko dusre URL pe bhej deta hai. Form submit ke baad kisi aur page pe le jana.

Example code:
app.get('/', (req, res) => {
  res.send('Hello');           // plain text
});

app.get('/users', (req, res) => {
  res.render('users', { data: userList }); // EJS template render
});

app.get('/api/users', (req, res) => {
  res.json({ users: userList }); // pure data, no HTML
});

app.post('/login', (req, res) => {
  res.redirect('/dashboard');  // page change
});

-----------------------------------
SIMPLE WAY TO REMEMBER
-----------------------------------
GET/POST/PUT/DELETE = "request kis type ki hai" (client se server tak)
send/render/json/redirect = "response mein kya bhejna hai" (server se client tak)

Ek route mein dono hote hain saath, jaise:
app.post('/signup', (req, res) => {
  ...jwt/bcrypt logic...
  res.redirect('/login')
})
Yaha POST request handle ho rahi hai, aur end mein redirect response ja raha hai.
